lw-for

Render a list of elements by iterating over an array with lw-for. Each iteration exposes the current item and an optional $index.

HTML 

   <div lw lw-for='item, $index in items'>$index+': '+item</div>
Javascript 

   class extends LWElement {
      items = ['one', 'two', 'three'];
   }
$index+': '+item

You could access DOM nodes for each element in a lw-for loop by calling elem.getDom() as long as typeof elem evaluates to object.

Keyed rows: lw-key

Without a key, rows are reused by position: when the array is reordered, each DOM node stays where it is and receives whichever item now sits at its index. Anything living on the node itself — focus, a CSS transition, text typed into an input — stays behind. Add lw-key to match rows to items by identity instead: the node then moves with its item. The key expression sees the loop variable and must yield defined, unique values.

HTML 

   <!-- rows reused by position -->
   <div lw-for='p in people'>...</div>

   <!-- rows follow their item -->
   <div lw-for='p in people' lw-key='p.id'>...</div>
Javascript 

   class extends LWElement {
      people = [
         { id: 1, name: 'Alice' },
         { id: 2, name: 'Bob' },
         { id: 3, name: 'Carol' },
      ];

      reversePeople() {
         this.people.reverse();
      }
   }

Try it: type into a box next to a name, then reverse the list. On the left (no key) the text stays at its position and ends up next to the wrong name; on the right (keyed) it travels with its person.

p.name
