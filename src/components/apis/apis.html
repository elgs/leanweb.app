globalThis.leanweb

The runtime's only footprint on the global scope — everything hangs off it.

updateComponents(...tagNames)

Re-render components by tag name; with no arguments, every component in the app. Only changed parts are written to the DOM.

eventBus

The shared LWEventBus instance — see below.

urlHash

Mirrors location.hash. Read it in expressions, assign it to navigate.

urlHashPath

The path part of the hash: for #/login?a=1 it reads #/login. Assigning it keeps the params.

urlHashParams

The params as an object: for #/login?a=1&a=2&c=3 it reads {a: ['1', '2'], c: '3'}. Assigning it rewrites the query.

runtimeVersion

The project's runtime version — the version chip in the hero renders it live.

LWElement

The base class. A Leanweb component is an HTMLElement with its directives wired up — override the hooks you need.

update(rootNode)

Re-render the component, or just a subtree when rootNode is given. Called for you on local events, model changes, settled async methods, and parent renders.

domReady()

Called once, after the first render is bound and evaluated.

inputReady()

Called when the parent's lw-input: values have arrived.

turnedOn() · turnedOff()

Called when the component's own lw-if restores it — or parks it.

urlHashChanged()

If defined, called on every hash change, and the component re-renders after it — the routing hook.

LWEventBus

A tiny pub/sub shared by every component at leanweb.eventBus.

addEventListener(eventName, callback)

Subscribe. The callback receives {eventName, data}; the returned listener is your unsubscribe ticket.

removeEventListener(listener)

Unsubscribe whatever addEventListener returned.

dispatchEvent(eventName, data)

Publish to every current subscriber.
