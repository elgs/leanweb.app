lw-for#

Repeat an element for every item of an array. The loop variable and an optional $index are in scope for every binding inside the row.


            <ul>
               <li lw lw-for='planet, $index in planets'>$index + 1 + '. ' + planet</li>
            </ul>
         

            class extends LWElement {
               planets = ['Mercury', 'Venus', 'Earth'];
            }
  • $index + 1 + '. ' + planet

Keyed rows with lw-key

Without a key, rows are reused by position: reorder the array and each DOM node stays put, receiving whichever item now sits at its index — anything living on the node itself (focus, a transition, text typed into an input) stays behind. Add lw-key to match rows to items by identity instead, so the node travels with its data. Keys must be defined and unique; for rows re-fetched from a server, key by a stable field like p.id.


            <button lw-on:click='people.reverse()'>Reverse</button>

            <!-- reused by position: contents rewritten in place -->
            <div lw-for='p in people'>
               <span lw>p.name</span> <input placeholder='type here'>
            </div>

            <!-- keyed: the row node moves with its person -->
            <div lw-for='p in people' lw-key='p.id'>
               <span lw>p.name</span> <input placeholder='type here'>
            </div>
         

            class extends LWElement {
               people = [
                  { id: 1, name: 'Ada' },
                  { id: 2, name: 'Grace' },
               ];
            }
